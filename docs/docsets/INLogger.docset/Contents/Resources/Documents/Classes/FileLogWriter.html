FileLogWriter

public final class FileLogWriter : LogWriter, @unchecked Sendable

A log writer specific for log files used by the Logger‘s pipeline.

A queue ensures the ACID principles for logs written to files by this writer instance. However, be aware that when creating two different instances of a FileLogWriter both have different queues and thus might still mess up when writing to the same file, therefore, each writer should be responsible only for their own files and not share the same file with other writer.

Any logs get written to the log file, but when a new log file is about to create then the old log file gets backed up. This makes the log file to rotate where we have a current log file and an old log file with a total of up to two log files for rotation.

	}
)

    Parameters

    fileName

    The name of the file where to write any log statements to.
    backupName

    The name of the backup file where to save the log file when a new gets created.
    folder

    An URL to the folder where to save the log and backup files. Defaults to the documents directory.
    redirectStderrToLogfile

    Set to true to redirect the standard error stream to the log file to make sure also any print and NSLog statements are logged from 3rd party libs. Be aware that this can muddle the log file because it can happen that the log writer is trying to write something to the file in the same time as the stderr stream is sending to the file. Therefore, that can cause log statements in the file being interrupted by other statement intersecting them. Also it’s only possible to direct the stderr to one log file, therefore, don’t set to true for more than one FileLogWriter instance. Defaults to false.
    internalErrorReporter

    A closure to which internal errors are reported. Defaultly this just prints the error via NSLog. Since we are part of the logger itself we can't use the logger to log an error related to the Logger, therefore, we are using this reporter to have at least a chance during development to find problems related to the FileLogWriter` and we can hook into this for UnitTests to verify any error messages.
